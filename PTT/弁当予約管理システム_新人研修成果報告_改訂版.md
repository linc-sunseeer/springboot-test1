# 1. システム概要

## 開発目的
- 社内のお弁当予約を簡単にする
- 紙の管理からデジタルに変える
- 予約状況をすぐに確認できる

## 使う人
- **一般ユーザー**：お弁当の予約・キャンセル
- **管理者**：メニュー登録・予約集計

# 6. システム画面紹介

## ユーザー画面
1. **ログイン画面** → メール・パスワードで認証
2. **メニュー一覧** → 日替わり弁当を表示
3. **予約画面** → カートで数量選択
4. **マイページ** → 予約履歴確認

## 管理者画面
1. **ダッシュボード** → 予約状況一目了然
2. **メニュー管理** → 画像付きで登録
3. **予約集計** → 日別・月別で表示
4. **ユーザー管理** → 一覧表示・削除

# 7. データベース設計

## 主要テーブル (6 つ)

| テーブル | 説明 |
|----------|------|
| users | ユーザー情報（名前・メール・パスワード） |
| admins | 管理者情報 |
| menus | 弁当メニュー（画像・価格・日付） |
| reservations | 予約情報（数量・状態） |
| email_tokens | メール認証トークン |
| system_settings | システム設定 |

# 8. 工夫した点

## セキュリティ
- パスワードはハッシュ化して保存
- JWT トークンで認証
- メール認証で本人確認

## 使いやすい設計
- 前後端分離で画面が速い
- カート機能で複数予約できる
- 画像アップロードでメニューが見やすい

## テスト
- 単体テスト 18 件実施
- H2 データベースでテスト環境
