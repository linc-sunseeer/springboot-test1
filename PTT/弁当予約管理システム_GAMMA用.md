# 1. システム概要

## 開発目的
- 社内での弁当予約業務を効率化
- 紙ベースの予約管理をデジタル化
- 予約状況のリアルタイム把握

## 対象ユーザー
- 一般ユーザー：弁当の予約・キャンセル
- 管理者：メニュー登録・予約集計

# 5. 基本設計（データベース）

## 主要テーブル

| テーブル名 | 説明 |
|------------|------|
| users | ユーザー情報 |
| menus | 弁当メニュー |
| reservations | 予約情報 |
| categories | メニューカテゴリ |

## users テーブル仕様
| 項目名 | データ型 | 説明 |
|--------|----------|------|
| id | BIGINT | ユーザーID（PK） |
| name | VARCHAR(100) | 氏名 |
| email | VARCHAR(255) | メールアドレス |
| password | VARCHAR(255) | パスワード |
| role | VARCHAR(50) | 権限（USER/ADMIN） |

# 6. 基本設計（画面遷移図）

## 画面遷移フロー

```
ログイン画面
    ↓
メニュー一覧画面
    ↓
予約確認画面
    ↓
予約完了画面
```

## 管理者画面
```
管理者ログイン
    ↓
ダッシュボード
    ↓
メニュー管理 / 予約集計 / ユーザー管理
```

---

# 7. 画面説明

## ログイン画面
- メールアドレスとパスワードで認証
- 新規登録リンクあり

## メニュー一覧画面
- 日替わりメニューを表示
- 予約ボタンで予約画面へ遷移

## 予約確認画面
- 選択したメニューの詳細表示
- 数量・配達先を入力

## 予約完了画面
- 予約内容の確認
- 予約番号の表示

# 8. テスト仕様書

## ログイン画面テスト

| テスト項目 | 入力値 | 期待結果 |
|------------|--------|----------|
| 正常系ログイン | 登録済みID/PW | ホーム画面へ遷移 |
| 異常系ログイン | 未登録ID | エラーメッセージ表示 |
| 空欄チェック | 空欄 | バリデーションエラー |

## 予約機能テスト

| テスト項目 | 入力値 | 期待結果 |
|------------|--------|----------|
| 正常系予約 | メニュー選択・数量入力 | 予約完了画面表示 |
| 二重予約防止 | 同一メニュー同時予約 | エラーメッセージ表示 |
| キャンセル | 予約済みのキャンセル | 予約状態がキャンセルに |
