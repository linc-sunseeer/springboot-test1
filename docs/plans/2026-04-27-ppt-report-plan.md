### 第 3 頁：システム概要
**開発目的**
- 社内のお弁当予約を簡単にする
- 紙の管理からデジタルに変える
- 予約状況をすぐに確認できる

**使う人**
- 一般ユーザー：お弁当を予約・キャンセル
- 管理者：メニュー登録・予約集計

### 第 5 頁：開発環境＆技術スタック
**バックエンド**
- Java 17
- Spring Boot 3.2.6
- MyBatis-Plus 3.5.7
- Spring Security + JWT
- Spring Mail

**フロントエンド**
- Vue.js 3 + Vite
- Axios (API 通信)
- Pinia (状態管理)

**その他**
- MySQL 8.0 / H2 (テスト)
- Docker
- Git / GitHub
- Lombok

> ⚠️ 修正點：原 PPT 寫的是 Spring Data JPA + Thymeleaf，實際是 MyBatis-Plus + Vue.js

### 第 10 頁：工夫した点・技術的な挑戦
**セキュリティ**
- パスワードはハッシュ化して保存
- JWT トークンで認証
- メール認証で本人確認

**使いやすい設計**
- 前後端分離で画面が速い
- カート機能で複数予約できる
- 画像アップロードでメニューが見やすい

**テスト**
- 単体テスト 18 件実施
- H2 データベースでテスト環境

**今後の改善点**
- 決済機能の追加
- モバイルアプリ対応
- パフォーマンス最適化
- テストカバレッジ向上

## 報告時のポイント

### 簡単な日本語で話すコツ
1. **短い文で話す**
   - ❌ 「Spring Security を使用して JWT 認証を実装しました」
   - ✅ 「ログインは JWT で安全にしました」

2. **漢字より平假名**
   - ❌ 「実装」→ ✅ 「作った」
   - ❌ 「確認」→ ✅ 「チェック」

3. **具体的な数字を使う**
   - 「3 ヶ月で作った」
   - 「18 個のテストを作った」
   - 「6 つのテーブルを使った」

### 質問されそうなこと
1. なぜ MyBatis-Plus を選んだ？
   → 書き方が簡単、クエリが作りやすい
2. JWT とは？
   → ログイン状態を保つ仕組み
3. メール認証はなぜ必要？
   → 本物のメールアドレスか確認するため
4. 一番苦労したことは？
   → JWT 認証の設定 / メール送信の設定
